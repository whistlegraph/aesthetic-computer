% ============ 1. THE FOUR REPOSITORIES ============

\section{四个代码仓库}

\ac{}并非从一开始就是单一代码仓库。它经历了四个连续的 GitHub 仓库的演变，每个仓库代表一个不同的架构阶段：

\begin{table}[h]
\small
\centering
\begin{tabular}{lr}
\toprule
\textbf{指标} & \textbf{数值} \\
\midrule
连续仓库数 & 4 \\
总提交次数（当前） & 11,314 \\
开发月数 & 38 \\
前身项目数 & 94 \\
\bottomrule
\end{tabular}
\end{table}

\subsection{system-ac（2021年8月--12月）}

\texttt{whistlegraph/system-ac} --- \emph{``我的虚拟计算机系统，用于加载和运行磁盘。''}

第一次提交于2021年8月10日，并立即从 Glitch~\citep{glitch2017} 合并而来，表明AC最初是一个 Glitch remix 项目。该仓库包含四个月内的38次提交，作者署名分别为``@jeffrey''、``Jeffrey Scudder''和``whistlegraph''——这是与代码同步进行的身份整合的痕迹。

初始架构非常精简：一个 \texttt{boot.js} 入口点、一个 \texttt{bios.js} 运行时，以及一系列模块库，包括 \texttt{disk.js}、\texttt{geo.js}、\texttt{graph.js}、\texttt{pen.js}、\texttt{speaker.js}，和一个支持正弦波与方波发生器的声音系统。隐喻从一开始就已确立：一台加载和运行\emph{磁盘}的计算机。

关键里程碑：基本节拍器和方波乐器（8月18日），三角形光栅化和3D环境（10月20日），首个动画记谱作品（10月30日），pen 从单例重构为类（11月28日）——一个关键的API设计决策。开发服务器运行在8080端口，使用自签名SSL证书；从第一天起就要求使用HTTPS，这可能是为了 WebRTC 和音频 API 的需要。

\subsection{disks-ac（2021年10月--12月）}

\texttt{whistlegraph/disks-ac} --- \emph{``一批与 `whistlegraph/system' 仓库兼容的磁盘。''}

这个配套仓库将创意内容与运行时分开存放。其 \texttt{package.json} 揭示了两者的耦合关系：\texttt{"system": "file:../system"}——磁盘将系统作为本地依赖引入。开发服务器运行在8081端口，比系统端口高一个。

最初的12张磁盘代表了AC的基础创意词汇：\texttt{prompt.js}（命令行文本输入——系统的正门）、\texttt{doodle.js}（自由绘画）、\texttt{plot.js}（基于网格的像素绘制）、\texttt{pull.js}（触摸交互形状）、\texttt{stage.js}（音乐记谱）、\texttt{starfield.js}（生成式动画）、\texttt{tracker.js}（可滚动的音序器）、\texttt{whistlegraph.js}（Whistlegraph 回放），以及四个实用/测试磁盘。另外保留了 \texttt{old/worker-disk.js}——表明基于 Web Worker 的磁盘执行模型曾被尝试过但很早就被放弃了。

\subsection{2022.aesthetic.computer（2021年12月--2022年12月）}

\texttt{whistlegraph/2022.aesthetic.computer} --- \emph{``进行美学计算。''}

2021年12月22日，系统和磁盘仓库被合并：\emph{``从先前仓库导入源代码，''}紧接着是\emph{``删除重复的磁盘。''}这是第一个单一代码仓库。标语从技术描述变为了创意宣言。

这个为期一年的仓库记录了AC从个人乐器向平台的转变：

\begin{itemize}
  \item \textbf{2022年1月2日}：``开始为 iPad 和 Apple Pencil 开发喷绘工具。''同日：``添加 iOS / macOS 原生适配层。''
  \item \textbf{1月5日}：``完成基本的 IPFS 演示''——很早就探索了 Web3/去中心化存储。
  \item \textbf{1月7日}：``将磁盘整合进系统''——两仓库架构在两周后就成了摩擦点。
  \item \textbf{1月10日}：``开始处理 prompt，连接键盘。''
  \item \textbf{1月11日}：``连接麦克风，开始视频代码路径。''
  \item \textbf{2022年10月}：Hypervisor 改造——作品现在可以即时切换而无需页面刷新。
  \item \textbf{10月24日}：``Meta Quest 2 键盘更新 / 更多HID工作''——VR头显支持。
  \item \textbf{2022年11月}：VR集成：``cadwand vr hello world，''3D画笔工具，GLTF/GLB导出。
  \item \textbf{2022年12月}：``完成以太坊登录/连接流程。''使用 Jamsocket 后端的会话服务器。
\end{itemize}

到年底，项目目录树包括：\texttt{system/}（Netlify）、\texttt{session-server/}（Jamsocket）、\texttt{socket-server/}、\texttt{piece-server/}、\texttt{thumbnail-server/}、\texttt{stream/}、\texttt{storage/}、\texttt{apple/}（iOS/macOS适配层）、\texttt{ethereum/}，以及 \texttt{digitpain.com/}。

\subsection{aesthetic-computer（2022年12月--至今）}

当前仓库。第一次提交：\emph{``Initial commit。''}第二次：\emph{``将所有先前源代码复制到新仓库；待办事项。''}第三次：\emph{``首次 Netlify 部署。''}2022年12月23日，35分钟内完成三次提交——在前一个仓库收到最后一次提交的同一天重新开始。

截至2026年3月：跨越38个月的\textbf{11,314次提交}。

% ============ 2. THE PREDECESSOR PROJECTS ============

\section{前身项目}

AC并非凭空出现。通过交叉参考两个 GitHub 账户（拥有50多个仓库的 \texttt{whistlegraph} 和拥有3个仓库的 \texttt{justanothersystem}）、一个 Dropbox 归档、一个109\,GB的遗留 DigitalOcean 云主机，以及作者的简历，我们确定了跨越2007--2020年的\textbf{94个前身项目}。

\subsection{绘画软件谱系}

AC的核心是一个绘画程序。其谱系如下：

\begin{itemize}
  \item \textbf{thePRBAT}（2011）--- 多边形复制位图创作工具。Flash/SWF。Ringling College 本科毕业论文工具。
  \item \textbf{JeffreyPaint}（2014）--- ``我的绘画程序。''JavaScript 网页应用 + Unity 移植 + macOS .dmg。
  \item \textbf{No Paint}（2014--2020）--- ``一个图像制作乐器。''约束条件：只能画，不能擦。六年间经历五次迭代：Rails 应用 $\rightarrow$ JS 适配层 $\rightarrow$ WebGL 原型 $\rightarrow$ iOS/Cordova $\rightarrow$ Vercel 站点。六个仓库。
  \item \textbf{Mood Engine}（2015）--- 原生C应用程序。``Maskware 的 MOOD 引擎。''
  \item \textbf{Traveller}（2015）--- Processing $\rightarrow$ 3个独立的 macOS .app 构建。
  \item \textbf{Finger Quilt}（2016）--- 使用 Swift 开发的 iOS 应用。以拼布为绘画。
  \item \textbf{Dot}（2017）--- ``一个点阵绘制应用。''直接预示了AC的 \texttt{plot} 磁盘。
\end{itemize}

作者还维护了一个网页绘画工具的精选目录（2017年，以 \texttt{weird-drawing-tools.glitch.me} 托管在 Glitch 上）——一份记录了数十个绘画程序的研究文档。这份目录代表了对``绘画软件即媒介''的有意识研究，为AC的设计提供了信息。

\subsection{表演软件谱系}

AC的实时、多人联机和表演功能可追溯到为``激进数字绘画''巡回演讲和 Goodiepal \& Pals 合作项目而构建的工具：

\begin{itemize}
  \item \textbf{Peter}（2017）--- 一个允许观众上传会过期的图像供 Julia Yerger 绘画使用的网页应用。实时观众参与。
  \item \textbf{border-notation}（2017）--- 为 GP\&PLS 开发的边框动画记谱程序。浏览器中的音乐记谱。
  \item \textbf{pmet-subs}（2017）--- Goodiepal 表演的实时字幕显示。
  \item \textbf{Shutter Rub}（2018）--- 在 Fikra Design Biennal 为 Knoth \& Renner 开发的工具。
  \item \textbf{pic.jeffreyheart89.com}（2017）--- ``数字绘画送达服务。''
\end{itemize}

\subsection{基础设施谱系}

即使AC的基础设施也有前身：

\begin{itemize}
  \item \textbf{live}（2016）--- ``自动重新加载浏览器。''一个 Ruby 实时重载服务器。AC当前的热重载 WebSocket 系统是其后继者。
  \item \textbf{cfg}（2013）--- 耶鲁 MFA 时期的 VimL 配置文件。最早的``配置即实践''。
  \item \textbf{netlify-functions-headless-chrome}（2022）--- Netlify 上的 Headless Chrome 截图。oven 截图服务的直接前身。
  \item \textbf{giphy-upload-proxy}（2017）--- 服务器端媒体处理经验。
\end{itemize}

\subsection{跨项目编程语言}

前身项目涵盖了对单个实践者而言异常广泛的语言范围：JavaScript（主导）、Ruby（Rails 时期）、Swift（iOS）、Java（Android）、C（Mood Engine）、Rust（Rect）、VimL（配置文件）、Processing（Traveller）、Flash/ActionScript（thePRBAT）、Lua（PICO-8）和 TypeScript。AC将这段多语言历史整合为以 JavaScript 为主的平台，以 KidLisp~\citep{scudder2026kidlisp} 作为其嵌入式语言。

% ============ 3. ARCHITECTURAL PATTERNS ============

\section{架构模式}

\subsection{磁盘/作品隐喻}

从 \texttt{system-ac} 的第一次提交开始，隐喻就是由``系统''通过``bios''加载的``磁盘''。这个借自个人计算历史的隐喻贯穿了所有四个仓库。从``磁盘''到``作品''的更名发生在2022年期间，将技术术语与艺术术语对齐。一件软件作品（a piece of software）。一首音乐作品（a piece of music）。一件艺术作品（a piece of art）。这次更名的意涵在配套论文~\citep{scudder2026pieces}中有所探讨。

\subsection{作为界面的提示符}

提示符——一个闪烁的文本光标，用户在此输入作品名称——出现在最早的 \texttt{disks-ac} 仓库中，作为 \texttt{prompt.js}。它后来发展成为 notepat 的键盘和AC的主要导航系统。提示符不是命令行；它是一个\emph{音乐性的}界面，在其中输入 \texttt{notepat} 这样的词是一种回忆和调用的行为，就像哼一段旋律来找到一首歌。

\subsection{双仓库到单一代码仓库}

系统/磁盘的分离仅持续了整整4个月就被合并了。这种分离反映了一种概念上的区分（运行时与内容），但在实践中证明不可行。合并又发生了两次：先是合并到 \texttt{2022.aesthetic.computer}，然后是当前仓库。每次整合都舍弃了 git 历史，转而追求一个干净的起点——这是一个有意的选择，优先考虑前进的动力而非考古的连续性，而这恰恰是使本次重建工作成为必要的原因。

% ============ 5. CONCLUSION ============

\section{结论}

\ac{}并非始于2021年8月10日的项目。它是十年创意软件实践的汇聚——从 thePRBAT（2011）到 No Paint 的五次迭代（2014--2020），经由为 Goodiepal \& Pals 开发的表演工具，经由作者能找到的每一个网页绘画工具的精选目录，经由一个 Rust 矩形游戏、一个 PICO-8 绘画卡带和一个 Pebble 智能手表实验，经由65场以上的演讲表演，论证数字工具是绘画的下一个材料基础。

git 历史揭示了宣言无法揭示的：日常决策的积累，被放弃的路径（\texttt{worker-disk.js}，双仓库分离），汇聚（系统 + 磁盘 $\rightarrow$ 单一代码仓库），以及承载意义的更名（disk $\rightarrow$ piece，notepad $\rightarrow$ notepat，NFT $\rightarrow$ keep）。每一次提交都是 @jeffrey 创作过的最长画作中的一笔。

\begin{quote}
\emph{``用30分钟创作的东西（比如一个 whistlegraph！）可以永恒地存在。''}

--- @jeffrey，个人陈述：核心信念
\end{quote}
