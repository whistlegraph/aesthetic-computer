\begin{quote}
\small\noindent\textbf{Abstract.}
This paper traces the complete technical evolution of \ac{} (AC) through its git history, from a Glitch-hosted prototype in August 2021 to an 11,314-commit monorepo serving 337 interactive pieces. By examining four successive repositories, 50+ GitHub repos across two accounts, a 109\,GB legacy server archive, and a Dropbox archive spanning a decade of creative software practice, we reconstruct how a single artist-programmer's tools, prototypes, and creative instruments converged into a unified platform. The investigation reveals 94 predecessor projects (2007--2020) that directly inform AC's design philosophy: that software is a material for art-making, and that each tool an artist builds is a step toward a more complete instrument.
\end{quote}

% ============ 1. THE FOUR REPOSITORIES ============

\section{The Four Repositories}

\ac{} did not begin as a monorepo. It evolved through four successive GitHub repositories, each representing a distinct architectural phase:

\small
\centering
\begin{tabular}{lr}
\toprule
\textbf{Metric} & \textbf{Value} \\
\midrule
Successive repositories & 4 \\
Total commits (current) & 11,314 \\
Months of development & 38 \\
Predecessor projects & 94 \\
\bottomrule
\end{tabular}
\end{table}

\subsection{system-ac (Aug--Dec 2021)}

\texttt{whistlegraph/system-ac} --- \emph{``My virtual computer system for loading and running disks.''}

The first commit was made on August 10, 2021 and immediately merged from Glitch~\citep{glitch2017}, revealing that AC began as a Glitch remix. The repository contains 38 commits over four months, authored variously as ``@jeffrey,'' ``Jeffrey Scudder,'' and ``whistlegraph'' --- a trace of the identity consolidation happening in parallel with the code.

The initial architecture was minimal: a single \texttt{boot.js} entry point, a \texttt{bios.js} runtime, and a library of modules including \texttt{disk.js}, \texttt{geo.js}, \texttt{graph.js}, \texttt{pen.js}, \texttt{speaker.js}, and a sound system with sine and square wave generators. The metaphor was already established: a computer that loads and runs \emph{disks}.

Key milestones: basic metronome and square instrument (Aug~18), triangle rasterization and 3D environment (Oct~20), first animated notation piece (Oct~30), pen refactored from singleton to class (Nov~28) --- a pivotal API design decision. The dev server ran on port 8080 with self-signed SSL certs; HTTPS was required from day one, likely for WebRTC and audio APIs.

\subsection{disks-ac (Oct--Dec 2021)}

\texttt{whistlegraph/disks-ac} --- \emph{``A bunch of disks that are compatible with the `whistlegraph/system' repo.''}

This companion repository held creative content separately from the runtime. Its \texttt{package.json} reveals the coupling: \texttt{"system": "file:../system"} --- the disks imported system as a local dependency. The dev server ran on port 8081, one above system.

The 12 original disks represent AC's founding creative vocabulary: \texttt{prompt.js} (command-line text input --- the system's front door), \texttt{doodle.js} (free drawing), \texttt{plot.js} (grid-based pixel plotting), \texttt{pull.js} (touch-interactive shapes), \texttt{stage.js} (musical notation), \texttt{starfield.js} (generative animation), \texttt{tracker.js} (scrollable sequencer), \texttt{whistlegraph.js} (Whistlegraph playback), and four utility/test disks. Also preserved: \texttt{old/worker-disk.js} --- evidence that a Web Worker-based disk execution model was attempted and abandoned early.

\subsection{2022.aesthetic.computer (Dec 2021--Dec 2022)}

\texttt{whistlegraph/2022.aesthetic.computer} --- \emph{``Compute aesthetically.''}

On December 22, 2021, the system and disks repos were merged: \emph{``import sources from previous repositories,''} followed immediately by \emph{``rm disks duplicates.''} This was the first monorepo. The tagline changed from a technical description to a creative imperative.

This year-long repository captures AC's transformation from a personal instrument into a platform:

\begin{itemize}
  \item \textbf{Jan 2, 2022}: ``Start working on a spray drawing tool for iPad \& Apple Pencil.'' Same day: ``Add iOS / macOS native shim.''
  \item \textbf{Jan 5}: ``Get a basic IPFS demo working'' --- Web3/decentralized storage explored early.
  \item \textbf{Jan 7}: ``Consolidate disks into system'' --- the two-repo architecture was already a friction point after 2 weeks.
  \item \textbf{Jan 10}: ``Begin working on prompt, wiring up the keyboard.''
  \item \textbf{Jan 11}: ``Hook up microphone and start code path for video.''
  \item \textbf{Oct 2022}: Hypervisor revamp --- pieces can now transition instantly without page refresh.
  \item \textbf{Oct 24}: ``Meta Quest 2 keyboard updates / more HID work'' --- VR headset support.
  \item \textbf{Nov 2022}: VR integration: ``cadwand vr hello world,'' 3D wand tool, GLTF/GLB export.
  \item \textbf{Dec 2022}: ``Complete ethereum login / connect flow.'' Session server with Jamsocket backends.
\end{itemize}

By year's end the tree included: \texttt{system/} (Netlify), \texttt{session-server/} (Jamsocket), \texttt{socket-server/}, \texttt{piece-server/}, \texttt{thumbnail-server/}, \texttt{stream/}, \texttt{storage/}, \texttt{apple/} (iOS/macOS shim), \texttt{ethereum/}, and \texttt{digitpain.com/}.

\subsection{aesthetic-computer (Dec 2022--present)}

The current repository. First commit: \emph{``Initial commit.''} Second: \emph{``copy all previous sources into the new repo; todos.''} Third: \emph{``first netlify deploy.''} Three commits in 35 minutes on December 23, 2022 --- a fresh start on the same day the previous repo received its final commit.

As of March 2026: \textbf{11,314 commits} across 38 months.

% ============ 2. THE PREDECESSOR PROJECTS ============

\section{The Predecessor Projects}

AC did not emerge from nothing. Cross-referencing two GitHub accounts (\texttt{whistlegraph} with 50+ repos and \texttt{justanothersystem} with 3 repos), a Dropbox archive, a 109\,GB legacy DigitalOcean droplet, and the author's CV, we identify \textbf{94 predecessor projects} spanning 2007--2020.

\subsection{The Drawing Software Lineage}

AC is, at its core, a drawing program. The lineage:

\begin{itemize}
  \item \textbf{thePRBAT} (2011) --- Polygon Replicating Bitmap Authoring Tool. Flash/SWF. BFA thesis tool at Ringling College.
  \item \textbf{JeffreyPaint} (2014) --- ``My paint program.'' JavaScript web app + Unity port + macOS .dmg.
  \item \textbf{No Paint} (2014--2020) --- ``A picture-making instrument.'' The constraint: you can only paint, never erase. Five incarnations across six years: Rails app $\rightarrow$ JS shim $\rightarrow$ WebGL prototype $\rightarrow$ iOS/Cordova $\rightarrow$ Vercel site. Six repos.
  \item \textbf{Mood Engine} (2015) --- Native C application. ``Maskware's MOOD Engine.''
  \item \textbf{Traveller} (2015) --- Processing $\rightarrow$ 3 standalone macOS .app builds.
  \item \textbf{Finger Quilt} (2016) --- iOS app in Swift. Quilting-as-drawing.
  \item \textbf{Dot} (2017) --- ``A dot plotting application.'' Directly prefigures AC's \texttt{plot} disk.
\end{itemize}

The author also maintained a curated directory of web-based drawing tools (2017, hosted on Glitch as \texttt{weird-drawing-tools.glitch.me}) --- a research document cataloging dozens of drawing programs. This catalog represents the conscious study of drawing-software-as-medium that informed AC's design.

\subsection{The Performance Software Lineage}

AC's real-time, multiplayer, and performance features trace to tools built for the Radical Digital Painting lecture tour and Goodiepal \& Pals collaborations:

\begin{itemize}
  \item \textbf{Peter} (2017) --- A web app allowing audience members to upload expiring images for Julia Yerger to paint with. Real-time audience participation.
  \item \textbf{border-notation} (2017) --- Border animation notation program for GP\&PLS. Music notation in the browser.
  \item \textbf{pmet-subs} (2017) --- Real-time subtitle display for Goodiepal performance.
  \item \textbf{Shutter Rub} (2018) --- Tool for Knoth \& Renner at Fikra Design Biennal.
  \item \textbf{pic.jeffreyheart89.com} (2017) --- ``Digital painting delivery service.''
\end{itemize}

\subsection{The Infrastructure Lineage}

Even AC's infrastructure has predecessors:

\begin{itemize}
  \item \textbf{live} (2016) --- ``Reloads browser automatically.'' A Ruby live-reload server. AC's current hot-reload WebSocket system is the descendant.
  \item \textbf{cfg} (2013) --- VimL dotfiles from Yale MFA era. The first configuration-as-practice.
  \item \textbf{netlify-functions-headless-chrome} (2022) --- Headless Chrome screenshots on Netlify. Direct precursor to the oven screenshot service.
  \item \textbf{giphy-upload-proxy} (2017) --- Server-side media processing experience.
\end{itemize}

\subsection{Languages Across Projects}

The predecessor projects span an unusual range for a single practitioner: JavaScript (dominant), Ruby (Rails era), Swift (iOS), Java (Android), C (Mood Engine), Rust (Rect), VimL (dotfiles), Processing (Traveller), Flash/ActionScript (thePRBAT), Lua (PICO-8), and TypeScript. AC consolidates this polyglot history into a JavaScript-first platform with KidLisp~\citep{scudder2026kidlisp} as its embedded language.

% ============ 3. ARCHITECTURAL PATTERNS ============

\section{Architectural Patterns}

\subsection{The Disk/Piece Metaphor}

From the first \texttt{system-ac} commit, the metaphor was ``disks'' loaded by a ``system'' with a ``bios.'' This metaphor --- borrowed from personal computing history --- persists through all four repos. The rename from ``disk'' to ``piece'' happened during the 2022 period, aligning the technical term with the artistic one. A piece of software. A piece of music. A piece of art. The implications of this renaming are explored in a companion paper~\citep{scudder2026pieces}.

\subsection{The Prompt as Interface}

The prompt --- a blinking text cursor where users type piece names --- appears in the very first \texttt{disks-ac} repo as \texttt{prompt.js}. It later became notepat's keyboard and AC's primary navigation system. The prompt is not a command line; it is a \emph{musical} interface, where typing a word like \texttt{notepat} is an act of recall and invocation, like humming a melody to find a song.

\subsection{Two-Repo to Monorepo}

The system/disks split lasted exactly 4 months before being merged. The split reflected a conceptual distinction (runtime vs.\ content) that proved impractical. The merge happened twice more: into \texttt{2022.aesthetic.computer} and then into the current repo. Each consolidation discarded git history in favor of a clean start --- a deliberate choice to prioritize forward momentum over archaeological continuity, which is precisely what makes this reconstruction necessary.

% ============ 5. CONCLUSION ============

\section{Conclusion}

\ac{} is not a project that began on August 10, 2021. It is the convergence of a decade of creative software practice --- from thePRBAT (2011) through No Paint's five incarnations (2014--2020), through performance tools for Goodiepal \& Pals, through a curated catalog of every web-based drawing tool the author could find, through a Rust rectangle game and a PICO-8 drawing cart and a Pebble smartwatch experiment, through 65+ lecture performances on the thesis that digital tools are painting's next material ground.

The git history reveals what manifestos cannot: the daily accumulation of decisions, the abandoned paths (\texttt{worker-disk.js}, the two-repo split), the convergences (system + disks $\rightarrow$ monorepo), and the renames that carry meaning (disk $\rightarrow$ piece, notepad $\rightarrow$ notepat, NFT $\rightarrow$ keep). Every commit is a brushstroke in the longest painting @jeffrey has ever made.
