% ============ 1. WHY THIS AUDIT ============

\section{为何进行此审计}

\ac{} 论文系列~\citep{scudder2026ac, scudder2026kidlisp, scudder2026acos, scudder2026pieces, scudder2026notepat}对可及性、包容性和创意表达提出了主张。一个倡导普遍创意计算可及性的项目——将闲置笔记本电脑作为乐器、URL可寻址的程序、默认社交分发——必须审视其学术基础是否反映了它所倡导的多样性。

引用实践并非中立的。它们构成了一种知识经济：我们引用谁决定了谁被阅读、谁被雇用、谁获得资助，以及谁的思想被视为基础性的~\citep{benjamin2019race}。一个以白人、男性、美国/英国/欧洲作者为主导的论文系列，恰恰再生产了它声称要抵制的排斥。

这次审计不是一种指控，而是一种诊断。当前的引用分布反映了作者的学术训练、阅读历史以及所涉领域（媒体理论、计算机科学、人机交互）的结构。改变这一状况需要长期的刻意努力。

\subsection{这意味着什么}

引用语料库以白人、男性和欧美作者为绝对主导。这在创意计算论文中并不罕见——该领域的经典参考文献（Processing、Scratch、p5.js、Sonic Pi）均由位于 MIT、NYU、ITP 和 Cambridge 的团队创建。但"并不罕见"并不意味着"可以接受"。\ac{} 项目明确将自身定位为主流计算范式的替代方案；其引用应当反映替代性的声音。

% ============ 3. GAP ANALYSIS ============

\section{差距分析}

我们确定了引用语料库最薄弱的五个领域。

\subsection{创意编程中的女性}

创意编程和实时编程社区中有大量女性的重要工作，但我们的论文未予引用。关键人物包括：

\begin{itemize}
  \item \textbf{Joana Chicau}~\citep{chicau2021choreo}：编舞式编程、具身算法
  \item \textbf{Shelly Knotts}~\citep{knotts2015algorithmic}：网络音乐中的算法协作
  \item \textbf{Allison Parrish}~\citep{parrish2015programming}：计算诗歌、创意编程教育
  \item \textbf{Kate Compton}~\citep{compton2015tracery}：Tracery 生成文本语言
  \item \textbf{Lauren McCarthy}：已被引用（p5.js），但作为创意实践者的身份未被充分认可
  \item \textbf{Olivia Jack}：已被引用（Hydra），但作为拉丁美洲实时编程领域声音的身份未被充分认可
\end{itemize}

\subsection{黑人计算研究}

一个声称要民主化计算可及性的项目，必须与研究谁在历史上被排除在计算之外及其原因的学术成果进行对话：

\begin{itemize}
  \item \textbf{Ruha Benjamin}~\citep{benjamin2019race}：《技术之后的种族》考察了设计决策如何编码种族等级制度
  \item \textbf{Safiya Umoja Noble}~\citep{noble2018algorithms}：《压迫的算法》论述搜索引擎与种族偏见
  \item \textbf{Andr\'e Brock}~\citep{brock2020distributed}：《分布式黑人性》论述黑人数字文化与平台设计
  \item \textbf{Charlton McIlwain}~\citep{mcilwain2019black}：《黑人软件》揭示黑人计算的隐藏历史
\end{itemize}

\subsection{非西方理论}

媒体理论的引用完全来自欧洲传统（Kittler、McLuhan、Adorno）。非西方理论家提供了与 \ac{} 设计直接相关的框架：

\begin{itemize}
  \item \textbf{许煜（Yuk Hui）}~\citep{hui2016existence, hui2019recursion}：数字本体论、技术多样性——认为技术不必遵循单一的西方发展轨迹
  \item \textbf{Hito Steyerl}~\citep{steyerl2009poor}：《为贫乏图像辩护》论述流通、分辨率和数字物质性——与 \ac{} 的像素艺术美学和 URL 即媒介的方法直接相关
  \item \textbf{Kodwo Eshun}~\citep{eshun1998brilliant}：非洲未来主义声音小说——与 notepat 和音频合成工作相关
\end{itemize}

\subsection{原住民与去殖民计算}

操作系统论文的闲置硬件论点对计算主权具有深远影响，原住民学者已对此进行了论述：

\begin{itemize}
  \item \textbf{Lewis 等人}~\citep{lewis2018kin}：《与机器结亲》提出了人工智能和计算的原住民协议
  \item \textbf{Sasha Costanza-Chock}~\citep{costanza2020design}：《设计正义》论述社区主导的设计实践
  \item \textbf{Arturo Escobar}~\citep{escobar2018designs}：《为多元宇宙而设计》论述去殖民化的设计方法
\end{itemize}

\subsection{拉丁美洲创意编程}

Olivia Jack（Hydra，已被引用）是哥伦比亚人。更广泛的拉丁美洲创意编程社区代表性不足：

\begin{itemize}
  \item \textbf{Hernando Barrag\'an}~\citep{barragan2004wiring}：哥伦比亚人，Wiring 的创建者，Arduino 的直接前身——对物理计算具有奠基性意义，却鲜少被引用
  \item \textbf{C\'assia Bel}：巴西实时编码者和女性主义创意技术专家
  \item \textbf{Aar\'on Montoya-Moraga}：创意编程教育者，p5.js 社区组织者
\end{itemize}

% ============ 5. TARGETS ============

\section{目标与承诺}

\subsection{量化目标}

\begin{itemize}
  \item \textbf{论文系列中30\%的女性作者}（目前为11\%）
  \item \textbf{20\%的非美/英/欧作者}（目前为5\%）
  \item \textbf{15\%的黑人、原住民、拉美、亚裔学者}（目前为6\%）
  \item \textbf{每篇论文至少包含2篇女性作者的引用}
\end{itemize}

这些目标是理想的最低标准，而非上限。应通过添加相关学术成果来实现这些目标，而非删除现有引用或添加象征性的参考文献。

\subsection{过程承诺}

\begin{enumerate}
  \item \textbf{每次修订时进行多样性检查}：在提交任何论文之前，根据当前的引用人口统计重新进行审计
  \item \textbf{阅读清单维护}：研究合集中的引用多样性部分是一份动态文档——新著作在被发现和阅读后持续添加
  \item \textbf{全文获取}：在可能的情况下，在阅读文库中获取并存档被引用著作的开放获取版本，以实现真正的参与而非蜻蜓点水式的引用
  \item \textbf{诚实参与}：引用著作是因为它们能加强论证，而非因为它们能使电子表格更加多样化。如果一部著作被引用，它应当被阅读、理解并融入论文的推理之中
\end{enumerate}

% ============ 6. HONEST ASSESSMENT ============

\section{诚实的评估}

这次审计揭示了学术自我反思中的一个常见模式：作者意识到了问题，能够列举应当被引用的学者，并拥有追踪进展的技术基础设施——但尚未完成阅读。知道 Ruha Benjamin 写了《技术之后的种族》，并不等同于已经阅读了它、理解了它，并将其洞见融入了自己的设计实践。

意识与实践之间的差距正是这项工作的起点。研究合集中的阅读清单是对弥合这一差距的承诺。各论文目标是问责机制。审计本身——与它所批评的论文一同发布——是邀请读者按照作者为自己设定的标准来要求他。
