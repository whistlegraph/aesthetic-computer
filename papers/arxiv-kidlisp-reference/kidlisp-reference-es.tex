\small\noindent\textbf{Resumen.}
\kl{} es un dialecto Lisp mínimo para arte generativo con 118 funciones integradas en 12 categorías, sin funciones definidas por el usuario, sin recursión, sin E/S de archivos y sin redes. Es seguro por construcción: un programa \kl{} no puede escapar de su sandbox, hacer que su anfitrión falle ni acceder a recursos más allá de su lienzo. Este artículo sirve como referencia completa del lenguaje, documentando cada función integrada con ejemplos resaltados sintácticamente, el sistema de temporización (programación basada en fotogramas y segundos sin callbacks), el modelo de color (colores con nombre, RGB, gradientes, patrones), el sistema de composición por \texttt{\$}-códigos (incrustación de programas por referencia como mecanismo de abstracción principal), y el modelo de evaluación (reevaluación completa del AST cada fotograma con PRNG determinista). Presentamos \kl{} como un lenguaje diseñado alrededor de la \emph{omisión}: lo que no puede hacer es tan deliberado como lo que puede, y las restricciones producen un espacio creativo que 59 autores han usado para escribir 16.634 programas.
\end{quote}

\section{Sintaxis}

\kl{} usa sintaxis de S-expresiones~\citep{mccarthy1960recursive}. Cada expresión es un átomo (número, cadena, símbolo) o una lista encerrada en paréntesis:

\begin{lstlisting}
; Átomos
42                    ; número
"hello"               ; cadena
red                   ; símbolo (nombre de color)

; Listas (aplicación de funciones)
(wipe "navy")         ; limpiar pantalla azul marino
(ink "red")           ; establecer color de dibujo
(box 10 10 50 50)     ; dibujar rectángulo
(circle 64 64 30)     ; dibujar círculo
\end{lstlisting}

No hay operadores, ni notación infija, ni sintaxis especial más allá de paréntesis, cadenas, números y símbolos. La aritmética usa notación prefija:

\begin{lstlisting}
(+ 1 2)              ; => 3
(* 3 (+ 1 2))        ; => 9
(- 10 3)             ; => 7
(/ 100 4)            ; => 25
(% 17 5)             ; => 2 (módulo)
\end{lstlisting}

\section{Las 12 categorías}

\subsection{Gráficos (9 funciones)}

Las primitivas de dibujo operan en modo inmediato---cada llamada renderiza directamente al framebuffer.

\begin{lstlisting}
(wipe "black")        ; limpiar lienzo
(ink "cyan")          ; establecer color
(line 0 0 128 128)    ; línea diagonal
(box 10 10 40 40)     ; rectángulo
(circle 64 64 20)     ; círculo en centro
(tri 32 10 10 54 54 54) ; triángulo
(plot 64 64)          ; píxel individual
(flood 64 64)         ; relleno por inundación
(shape 10 10 50 10 30 50) ; polígono
\end{lstlisting}

\texttt{fill} y \texttt{outline} controlan si las formas son sólidas o con trazo (compatible con Processing):

\begin{lstlisting}
(fill)                ; formas rellenas
(ink "red")
(circle 64 64 30)     ; círculo rojo sólido

(outline)             ; formas con contorno
(ink "white")
(circle 64 64 30)     ; anillo blanco
\end{lstlisting}

\subsection{Transformaciones (11 funciones)}

Las transformaciones a nivel de píxel operan sobre el framebuffer después de dibujar:

\begin{lstlisting}
(scroll 1 0)          ; desplazar píxeles a la derecha
(zoom 1.01)           ; zoom lento
(spin 1)              ; rotar 1 grado
(blur 2)              ; desenfoque gaussiano
(contrast 1.2)        ; aumentar contraste
(suck 0.5)            ; desplazamiento radial
(sort)                ; ordenar píxeles por brillo
\end{lstlisting}

Las transformaciones se \emph{acumulan} entre fotogramas, produciendo animación sin gestión de estado explícita:

\begin{lstlisting}
(wipe "black")
(ink "white")
(circle 64 64 10)     ; dibujar círculo
(spin 2)              ; rotar 2 grados/fotograma
(zoom 1.005)          ; crecer ligeramente/fotograma
; Resultado: círculo en espiral, creciendo
\end{lstlisting}

\subsection{Matemáticas (14 funciones)}

\begin{lstlisting}
(sin (* frame 0.05))  ; onda sinusoidal
(cos (* frame 0.03))  ; onda cosinusoidal
(random)              ; 0-255
(random 10)           ; 0-9
(random 5 15)         ; 5-14
(wiggle 10)           ; aleatorio +/-5
(floor 3.7)           ; => 3
(abs -5)              ; => 5
(sqrt 16)             ; => 4
(min 3 7 1)           ; => 1
(max 3 7 1)           ; => 7
\end{lstlisting}

\texttt{random} usa un PRNG con semilla inicializada desde el código corto del programa, haciendo toda la salida generativa \textbf{determinista}: el mismo programa siempre produce la misma imagen.

\subsection{Color (19+ colores con nombre)}

Los colores pueden especificarse como nombres, valores RGB o cadenas de gradiente:

\begin{lstlisting}
(ink "red")           ; color con nombre
(ink 255 0 0)         ; RGB
(ink 255 0 0 128)     ; RGBA (semi-transparente)
(ink "fade:red-blue") ; gradiente horizontal
(ink (fade "cyan" "magenta" "vertical"))
(ink "rainbow")       ; arcoíris repetitivo
(ink "zebra")         ; patrón a rayas
\end{lstlisting}

Un nombre de color solo en la primera línea establece el fondo:

\begin{lstlisting}
"navy"
(ink "gold")
(circle 64 64 30)
; Fondo azul marino, círculo dorado
\end{lstlisting}

\subsection{Sistema y pantalla (9 funciones)}

\begin{lstlisting}
(def w width)         ; ancho de pantalla
(def h height)        ; alto de pantalla
frame                 ; número de fotograma actual
time                  ; segundos transcurridos

(resolution "half")   ; media resolución
(resolution 64 64)    ; explícito 64x64
(fps 30)              ; establecer tasa de fotogramas
\end{lstlisting}

\subsection{Flujo de control (7 funciones)}

\begin{lstlisting}
; Definición de variable
(def x 50)
(def y (+ 10 (* frame 0.5)))

; Condicional
(if (> frame 60)
  (ink "red")
  (ink "blue"))

; Bucle
(repeat 10 i
  (circle (* i 12) 64 5))

; Ejecutar una vez (no cada fotograma)
(once (wipe "black"))

; Vinculación local
(let ((cx 64) (cy 64))
  (circle cx cy 30))
\end{lstlisting}

El bucle \texttt{repeat} vincula su variable contadora (\texttt{i} arriba) de 0 a $n-1$. Los repeats anidados usan nombres de variables diferentes:

\begin{lstlisting}
(repeat 8 i
  (repeat 8 j
    (box (* i 16) (* j 16) 14 14)))
; Cuadrícula 8x8 de cuadrados
\end{lstlisting}

\subsection{Temporización}

El sistema de temporización de \kl{} usa \emph{programación en línea} en lugar de callbacks. Las expresiones de tiempo envuelven bloques de código:

\begin{lstlisting}
; Temporización basada en fotogramas
(0 (wipe "black"))    ; cada fotograma
(1 (spin 5))          ; cada 2do fotograma
(30f (ink "red"))     ; después de 30 fotogramas

; Temporización basada en segundos
(1s (wipe "red"))     ; después de 1 segundo
(2.5s (ink "gold"))   ; después de 2,5 segundos
(1s! (write "!" 10 10)) ; ejecutar una vez a 1s

; Intervalos repetitivos
(0.5s... (spin 45))   ; cada 0,5 segundos
(1s... (scroll 1 0))  ; cada segundo
\end{lstlisting}

Esto elimina la necesidad de \texttt{setTimeout}, \texttt{setInterval} o callbacks de fotograma de animación. El tiempo es una \emph{propiedad de la expresión}, no una función llamada sobre ella.

\subsection{Texto (4 funciones)}

\begin{lstlisting}
(write "hola" 10 20)  ; texto en posición
(write "centrado"     ; texto centrado
  (/ width 2) (/ height 2))
(type "escrito" 10 10) ; efecto máquina de escribir
\end{lstlisting}

\subsection{Entrada (3 funciones)}

\begin{lstlisting}
; Posición y estado del puntero/ratón
(if pen
  (circle pen.x pen.y 5))

; Entrada táctil
(if hand
  (line hand.x hand.y 64 64))
\end{lstlisting}

\subsection{Audio (6 funciones)}

\begin{lstlisting}
; Entrada de micrófono
(def vol (mic))       ; amplitud 0-255
(circle 64 64 vol)    ; reactivo al sonido

; Reproducción de melodía
(melody "C4 E4 G4 C5")

; Armónicos de sobretonos
(overtone 440 3)      ; 3er armónico de A4
\end{lstlisting}

\subsection{Gráficos 3D (8 funciones)}

\begin{lstlisting}
(cube "mycube")       ; crear cubo 3D
(form "mycube"
  (move 0 0 -5)       ; posición
  (scale 2)           ; tamaño
  (spin (* frame 0.5) 1 1 0)) ; rotar
\end{lstlisting}

\subsection{Composición vía \texttt{\$}-códigos}

La característica más distintiva de \kl{} es su sistema de composición. En lugar de funciones definidas por el usuario, los programas referencian otros programas por su código corto:

\begin{lstlisting}
; Incrustar otro programa como capa
($cow)                ; ejecutar programa "cow"
($nece 10 20)         ; pasar parámetros

; Composición multicapa
(wipe "black")
($cow)                ; capa de fondo
($faim)               ; capa de frente

; Encadenar múltiples programas
(resolution 64 64)
($zobi)
(spin 1)
($grel)
\end{lstlisting}

Los programas se obtienen de MongoDB, se almacenan en caché en RAM e IndexedDB, y se direccionan por contenido vía SHA-256. El sistema de \texttt{\$}-códigos reemplaza funciones definidas por usuario, módulos y bibliotecas con un único mecanismo social: \emph{encontrar un programa que alguien hizo, referenciarlo por código, componer con él}.

\section{Modelo de evaluación}

\kl{} reevalúa su \textbf{AST completo cada fotograma}. No hay estado retenido más allá de las variables explícitamente definidas. Las variables \texttt{frame} y \texttt{time} se incrementan automáticamente. Las transformaciones como \texttt{zoom}, \texttt{scroll} y \texttt{spin} se acumulan en el framebuffer.

Este modelo elimina la división \texttt{setup()}/\texttt{draw()} familiar de Processing~\citep{reas2007processing}. Cada línea de un programa \kl{} se ejecuta cada fotograma. La animación emerge de expresiones dependientes del fotograma:

\begin{lstlisting}
(wipe "black")
(ink "white")
; Círculo orbita el centro usando sin/cos
(circle
  (+ 64 (* 30 (cos (* frame 0.03))))
  (+ 64 (* 30 (sin (* frame 0.03))))
  8)
\end{lstlisting}

\subsection{Aleatoriedad determinista}

\texttt{random} tiene semilla del código corto del programa. El mismo programa produce salida idéntica en cada dispositivo, cada vez. Esto hace que los programas \kl{} sean \emph{arte generativo reproducible}: el código \emph{es} la obra de arte, no solo las instrucciones para generarla.

\subsection{Modo caos}

La entrada inválida---texto sin sentido, paréntesis desbalanceados, caracteres aleatorios---no produce mensajes de error. En su lugar, \kl{} detecta ratios de reconocimiento bajos ($<30\%$ palabras conocidas) y ratios de caracteres especiales altos ($>50\%$) y renderiza salida visual artística. El usuario nunca ve ``SyntaxError.'' Ve colores.

\section{Diseño por omisión}

\kl{} deliberadamente carece de:

\begin{itemize}
  \item \textbf{Funciones definidas por usuario}: reemplazadas por composición de \texttt{\$}-códigos
  \item \textbf{Recursión}: previene bucles infinitos y desbordamientos de pila
  \item \textbf{E/S de archivos}: el lienzo es la única salida
  \item \textbf{Redes}: los programas no pueden obtener datos externos
  \item \textbf{Manipulación de cadenas}: sin \texttt{concat}, \texttt{split}, \texttt{replace}
  \item \textbf{Mensajes de error}: reemplazados por modo caos
  \item \textbf{Import/require}: reemplazados por \texttt{\$}-códigos
  \item \textbf{Objetos/clases}: los datos son números, cadenas y listas
\end{itemize}

Cada omisión es una decisión de diseño. La ausencia de funciones definidas por usuario fuerza la composición a través de la capa social. La ausencia de mensajes de error significa que los principiantes nunca encuentran un muro de texto rojo. La ausencia de redes significa que cada programa es autocontenido y seguro de ejecutar.

Esto sigue el principio de que las restricciones de un lenguaje definen su espacio creativo más que sus funciones~\citep{papert1980mindstorms, compton2015casual}. \kl{} no es un lenguaje de propósito general que casualmente hace arte. Es un lenguaje de arte que \emph{no puede} hacer nada más.

\section{Ejemplo: programa completo}

Un programa \kl{} completo que dibuja un campo de partículas reactivo al sonido con movimiento orbital:

\begin{lstlisting}
"black"
(def vol (+ 5 (* (mic) 0.2)))
(repeat 50 i
  (def angle (* i 0.1256))
  (def r (+ 20 (* vol (sin (+ (* frame 0.02) angle)))))
  (def cx (+ 64 (* r (cos (+ angle (* frame 0.01))))))
  (def cy (+ 64 (* r (sin (+ angle (* frame 0.01))))))
  (ink (+ 100 (* i 3)) 50 (+ 150 (* i 2)))
  (circle cx cy (+ 1 (random 3))))
(blur 1)
(zoom 1.002)
\end{lstlisting}

Este programa: establece un fondo negro, lee la amplitud del micrófono, dibuja 50 partículas en trayectorias orbitales moduladas por sonido, las colorea con un gradiente calculado, aplica un ligero desenfoque, y lentamente acerca el zoom. Se ejecuta a 60 FPS. Tiene 12 líneas. No requiere importaciones, ni función de setup, ni bucle de animación, ni listeners de eventos.
