\begin{quote}
\small\noindent\textbf{Abstrakt.}
\kl{} er en minimal Lisp-dialekt til generativ kunst med 118 indbyggede funktioner fordelt på 12 kategorier, ingen brugerdefinerede funktioner, ingen rekursion, ingen fil-I/O og intet netværk. Det er sikkert ved konstruktion: et \kl{}-program kan ikke bryde ud af sin sandbox, crashe sin vært eller tilgå ressourcer ud over sit lærred. \section{Syntaks}

\kl{} bruger S-udtrykssyntaks~\citep{mccarthy1960recursive}. Ethvert udtryk er enten et atom (tal, streng, symbol) eller en liste omsluttet af parenteser:

\begin{lstlisting}
; Atomer
42                    ; tal
"hello"               ; streng
red                   ; symbol (farvenavn)

; Lister (funktionsapplikation)
(wipe "navy")         ; ryd skærm marineblå
(ink "red")           ; sæt tegnefarve
(box 10 10 50 50)     ; tegn rektangel
(circle 64 64 30)     ; tegn cirkel
\end{lstlisting}

Der er ingen operatorer, ingen infiks-notation, ingen speciel syntaks ud over parenteser, strenge, tal og symboler. Aritmetik bruger præfiks-notation:

\begin{lstlisting}
(+ 1 2)              ; => 3
(* 3 (+ 1 2))        ; => 9
(- 10 3)             ; => 7
(/ 100 4)            ; => 25
(% 17 5)             ; => 2 (modulo)
\end{lstlisting}

\section{De 12 kategorier}

\subsection{Grafik (9 funktioner)}

Tegneprimitiver opererer i immediate-mode---hvert kald renderer direkte til framebufferen.

\begin{lstlisting}
(wipe "black")        ; ryd lærred
(ink "cyan")          ; sæt farve
(line 0 0 128 128)    ; diagonal linje
(box 10 10 40 40)     ; rektangel
(circle 64 64 20)     ; cirkel i centrum
(tri 32 10 10 54 54 54) ; trekant
(plot 64 64)          ; enkelt pixel
(flood 64 64)         ; oversvømmelsesfyld
(shape 10 10 50 10 30 50) ; polygon
\end{lstlisting}

\texttt{fill} og \texttt{outline} kontrollerer, om former er udfyldte eller stregede:

\begin{lstlisting}
(fill)                ; udfyldte former
(ink "red")
(circle 64 64 30)     ; massiv rød cirkel

(outline)             ; stregede former
(ink "white")
(circle 64 64 30)     ; hvid ring
\end{lstlisting}

\subsection{Transformationer (11 funktioner)}

Pixelniveau-transformationer opererer på framebufferen efter tegning:

\begin{lstlisting}
(scroll 1 0)          ; flyt pixels til højre
(zoom 1.01)           ; langsom zoom ind
(spin 1)              ; rotér 1 grad
(blur 2)              ; gaussisk sløring
(contrast 1.2)        ; øg kontrast
(suck 0.5)            ; radial forskydning
(sort)                ; sortér pixels efter lysstyrke
\end{lstlisting}

Transformationer \emph{akkumulerer} på tværs af frames, hvilket producerer animation uden eksplicit tilstandsstyring.

\subsection{Matematik (14 funktioner)}

\begin{lstlisting}
(sin (* frame 0.05))  ; sinusbølge
(cos (* frame 0.03))  ; cosinusbølge
(random)              ; 0-255
(random 10)           ; 0-9
(random 5 15)         ; 5-14
(wiggle 10)           ; tilfældig +/-5
(floor 3.7)           ; => 3
(abs -5)              ; => 5
(sqrt 16)             ; => 4
(min 3 7 1)           ; => 1
(max 3 7 1)           ; => 7
\end{lstlisting}

\texttt{random} bruger en seedet PRNG initialiseret fra programmets korte kode, hvilket gør alt generativt output \textbf{deterministisk}: det samme program producerer altid det samme visuelle.

\subsection{Farve (19+ navngivne farver)}

\begin{lstlisting}
(ink "red")           ; navngivet farve
(ink 255 0 0)         ; RGB
(ink 255 0 0 128)     ; RGBA (semi-transparent)
(ink "fade:red-blue") ; horisontal gradient
(ink (fade "cyan" "magenta" "vertical"))
(ink "rainbow")       ; gentagende regnbue
(ink "zebra")         ; stribet mønster
\end{lstlisting}

\subsection{System og display (9 funktioner)}

\begin{lstlisting}
(def w width)         ; skærmbredde
(def h height)        ; skærmhøjde
frame                 ; nuværende framenummer
time                  ; sekunder forløbet

(resolution "half")   ; halv opløsning
(resolution 64 64)    ; eksplicit 64x64
(fps 30)              ; sæt framerate
\end{lstlisting}

\subsection{Kontrolflow (7 funktioner)}

\begin{lstlisting}
; Variabeldefinition
(def x 50)
(def y (+ 10 (* frame 0.5)))

; Betinget
(if (> frame 60)
  (ink "red")
  (ink "blue"))

; Løkke
(repeat 10 i
  (circle (* i 12) 64 5))

; Eksekvér én gang (ikke hvert frame)
(once (wipe "black"))

; Lokal binding
(let ((cx 64) (cy 64))
  (circle cx cy 30))
\end{lstlisting}

\subsection{Timing}

\kl{}'s timingsystem bruger \emph{inline-planlægning} snarere end callbacks:

\begin{lstlisting}
; Frame-baseret timing
(0 (wipe "black"))    ; hvert frame
(1 (spin 5))          ; hvert 2. frame
(30f (ink "red"))     ; efter 30 frames

; Sekundbaseret timing
(1s (wipe "red"))     ; efter 1 sekund
(2.5s (ink "gold"))   ; efter 2,5 sekunder
(1s! (write "!" 10 10)) ; affyr én gang ved 1s

; Gentagende intervaller
(0.5s... (spin 45))   ; hvert 0,5 sekund
(1s... (scroll 1 0))  ; hvert sekund
\end{lstlisting}

Dette eliminerer behovet for \texttt{setTimeout}, \texttt{setInterval} eller animationsframe-callbacks. Tid er en \emph{egenskab ved udtrykket}, ikke en funktion kaldt på det.

\subsection{Tekst (4 funktioner)}

\begin{lstlisting}
(write "hello" 10 20) ; tekst ved position
(write "centreret"    ; centreret tekst
  (/ width 2) (/ height 2))
(type "typed" 10 10)  ; skrivemaskineeffekt
\end{lstlisting}

\subsection{Input (3 funktioner)}

\begin{lstlisting}
; Pen/museposition og -tilstand
(if pen
  (circle pen.x pen.y 5))

; Berøringsinput
(if hand
  (line hand.x hand.y 64 64))
\end{lstlisting}

\subsection{Lyd (6 funktioner)}

\begin{lstlisting}
; Mikrofoninput
(def vol (mic))       ; amplitude 0-255
(circle 64 64 vol)    ; lydreaktiv

; Melodiafspilning
(melody "C4 E4 G4 C5")

; Overtonharmonikker
(overtone 440 3)      ; 3. harmonik af A4
\end{lstlisting}

\subsection{3D-grafik (8 funktioner)}

\begin{lstlisting}
(cube "mycube")       ; opret 3D-kube
(form "mycube"
  (move 0 0 -5)       ; position
  (scale 2)           ; størrelse
  (spin (* frame 0.5) 1 1 0)) ; rotér
\end{lstlisting}

\subsection{Komposition via \texttt{\$}-koder}

Den mest karakteristiske funktion i \kl{} er dens kompositionssystem. I stedet for brugerdefinerede funktioner refererer programmer til andre programmer via deres korte kode:

\begin{lstlisting}
; Indlejr et andet program som et lag
($cow)                ; eksekvér program "cow"
($nece 10 20)         ; send parametre

; Flerlagskomposition
(wipe "black")
($cow)                ; baggrundsslag
($faim)               ; forgrundslag
\end{lstlisting}

Programmer hentes fra MongoDB, caches i RAM og IndexedDB og indholdsadresseres via SHA-256. \texttt{\$}-kodesystemet erstatter brugerdefinerede funktioner, moduler og biblioteker med en enkelt social mekanisme: \emph{find et program nogen har lavet, referér det via kode, komponér med det}.

\section{Evalueringsmodel}

\kl{} reevaluerer sit \textbf{hele AST hvert frame}. Der er ingen bevaret tilstand ud over eksplicit definerede variabler. \texttt{frame}- og \texttt{time}-variablerne inkrementeres automatisk. Transformationer som \texttt{zoom}, \texttt{scroll} og \texttt{spin} akkumulerer i framebufferen.

Denne model eliminerer \texttt{setup()}/\texttt{draw()}-opdelingen kendt fra Processing~\citep{reas2007processing}. Enhver linje i et \kl{}-program eksekverer hvert frame.

\subsection{Deterministisk tilfældighed}

\texttt{random} er seedet fra programmets korte kode. Det samme program producerer identisk output på enhver enhed, hver gang. Dette gør \kl{}-programmer til \emph{reproducerbar generativ kunst}.

\subsection{Kaostilstand}

Ugyldig input producerer ikke fejlmeddelelser. I stedet detekterer \kl{} lave genkendelsesforhold og renderer kunstnerisk visuelt output. Brugeren ser aldrig ``SyntaxError.'' De ser farver.

\section{Design via udeladelse}

\kl{} mangler bevidst:

\begin{itemize}
  \item \textbf{Brugerdefinerede funktioner}: erstattet af \texttt{\$}-kode komposition
  \item \textbf{Rekursion}: forhindrer uendelige løkker og stakoverløb
  \item \textbf{Fil-I/O}: lærredet er det eneste output
  \item \textbf{Netværk}: programmer kan ikke hente eksterne data
  \item \textbf{Strengmanipulation}: ingen \texttt{concat}, \texttt{split}, \texttt{replace}
  \item \textbf{Fejlmeddelelser}: erstattet af kaostilstand
  \item \textbf{Import/require}: erstattet af \texttt{\$}-koder
  \item \textbf{Objekter/klasser}: data er tal, strenge og lister
\end{itemize}

Hver udeladelse er en designbeslutning. Fraværet af brugerdefinerede funktioner tvinger komposition gennem det sociale lag. Fraværet af fejlmeddelelser betyder, at begyndere aldrig møder en mur af rød tekst. Fraværet af netværk betyder, at ethvert program er selvstændigt og sikkert at køre.

Dette følger princippet om, at et sprogs begrænsninger definerer dets kreative rum mere end dets funktioner~\citep{papert1980mindstorms, compton2015casual}. \kl{} er ikke et generelt sprog, der tilfældigvis laver kunst. Det er et kunstsprog, der \emph{ikke kan} gøre andet.

\section{Eksempel: Komplet program}

Et komplet \kl{}-program, der tegner et lydreaktivt partikelfelt med orbital bevægelse:

\begin{lstlisting}
"black"
(def vol (+ 5 (* (mic) 0.2)))
(repeat 50 i
  (def angle (* i 0.1256))
  (def r (+ 20 (* vol (sin (+ (* frame 0.02) angle)))))
  (def cx (+ 64 (* r (cos (+ angle (* frame 0.01))))))
  (def cy (+ 64 (* r (sin (+ angle (* frame 0.01))))))
  (ink (+ 100 (* i 3)) 50 (+ 150 (* i 2)))
  (circle cx cy (+ 1 (random 3))))
(blur 1)
(zoom 1.002)
\end{lstlisting}

Dette program: sætter en sort baggrund, læser mikrofonamplitude, tegner 50 partikler i orbitale baner moduleret af lyd, farver dem med en beregnet gradient, anvender en let sløring og zoomer langsomt ind. Det kører ved 60 FPS. Det er 12 linjer langt. Det kræver ingen imports, ingen setup-funktion, ingen animationsløkke og ingen hændelseslyttere.
