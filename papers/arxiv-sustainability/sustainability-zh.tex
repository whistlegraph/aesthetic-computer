创意计算工具——Processing、p5.js、Sonic Pi、Pure Data、Hydra、TidalCycles、openFrameworks、Scratch 及其同类——被数百万人使用。它们的创造者通常没有人为其付费，或者由支持与工具无关的因素为条件的机构付费。本文调查了28位开源创意工具作者的资金历史，识别出六种可持续性模式：\emph{学术补贴}（大学薪水资助无偿的工具工作）、\emph{企业赞助}（公司为利益一致而付费）、\emph{个人牺牲}（储蓄、退休金、健康）、\emph{社区众筹}（Patreon、Open Collective、Ko-fi）、\emph{意外之财}（如 NFT 捐赠等不可预测的一次性事件），以及\emph{完全没有模式}（残疾补助金、无家可归、死亡）。我们发现，工具创建到首次获得可持续资金之间的中位间隔为8年，倦怠无论资金水平如何都会影响创作者，而使用最广泛的工具并非获得最好资助的工具。我们将这些发现与作者自己构建 \ac{} 的经验进行交叉参照——一个拥有2,800+注册用户、93,000+会话和16,000+社区编写程序的平台——在没有该领域每个可比工具都曾拥有的机构保护伞的情况下，历经五年的开发。我们论证，创意计算领域对隐性补贴——来自大学、来自配偶、来自储蓄账户、来自健康——的依赖不是一种资金模式，而是资金模式的缺失。
\end{quote}

\section{引言}

Lauren McCarthy 于2013年创建了 p5.js，此后十年每周在 UCLA 教职之外花费10--20小时无偿维护~\citep{mccarthy2023making}。Rich Hickey 花了2.5年退休储蓄来构建 Clojure，在任何人为此付费之前~\citep{hickey2020history}。Samuel Aaron 于2019年被 Cambridge 裁员，不得不"100\%专注于财务生存"，将 Sonic Pi 开发降至"蜗牛速度"~\citep{aaron2016sonic}。\_why the lucky stiff——Ruby 最受爱戴的贡献者之一——在被人肉搜索后删除了他的全部数字存在，从未为他的工作收到过一分钱。

Terry A. Davis 花了14.7年构建 TempleOS——一个完整的64位操作系统，包含自定义编程语言、编译器、内核、编辑器和游戏——121,176行代码，完全独自一人，靠社会保障残疾金资助，住在父母家中。当这一住所于2017年失去后，他住在一辆面包车里。2018年8月11日，他在 Oregon 州 The Dalles 被火车撞击身亡。年仅48岁。他构建的系统被拿来与 Xerox Alto 和 Plan 9 相比较。它的创造者死于无家可归。

这些不是边缘案例。它们是开源创意工具开发的正常状况。工具在运作。它们的创造者，往往不在。

本文调查了跨越四十年的28位创意工具作者，从 Larry Wall 1987年创建 Perl 到作者正在进行的 \ac{}~\citep{scudder2026ac} 开发。对于每位作者，我们识别：谁为工作付费、付费相对于创建何时开始、创作者牺牲了什么，以及资金安排是否延续。结果是一个可持续性模式的分类——以及它们失败的记录。

\section{五种模式}

\subsection{学术补贴}

创意计算工具最常见的资金模式根本不是资金模式：它是一份支付教学和研究费用的大学薪水，工具开发作为被归类为"服务"或"创作实践"的无偿劳动而进行。

补贴是真实的，但隐藏的。McCarthy 每周10--20小时的无偿 p5.js 维护之所以可能，是因为 UCLA 为她的其他工作支付了薪水。Puckette 在失去对 Max/MSP 的商业许可控制后创建了 Pure Data；他的 UCSD 教职资助了重建。Wang 的 ChucK 之所以存在，是因为 Stanford CCRMA 付费让他做研究。在每个案例中，工具都是学术生涯的副产品，而非其目的。

该模式有一个关键脆弱性：当学术职位结束时，工具就会饿死。Aaron 2019年从 Cambridge 被裁是最清楚的案例——当大学撤回支持后，Sonic Pi 开发崩溃了。Maeda 离开 MIT 实际上杀死了 Design By Numbers；Processing 之所以存活，只是因为他的学生将其独立出来。

Van Rossum 的轨迹是我们调查中获得企业支持最多的：CWI、CNRI、Google（50\% Python 时间）、Dropbox、Microsoft。每个雇主都付费让他在 Python 上工作。然而，即使有持续的企业支持，围绕 PEP 572 的社区毒性也驱使他在29年后辞去 BDFL 职位："人们上社交媒体说了一些真正伤害我个人的话。"

Wall 的 O'Reilly 赞助在2002年左右结束。此后他"正式失业了将近五年"，在胃部手术和长期疾病中继续开发 Perl~6。赞助者离开了；创作者留了下来。

Czaplicki 是对赞助为何不够的最清晰表达者。他的 Strange Loop 演讲~\citep{czaplicki2018hard, czaplicki2023economics}描述了基于捐赠的资金的结构性不可持续性，以及耗尽维护者精力的"你为什么不直接……"要求。他刻意限制了 Elm 的增长，而非不可持续地扩展。

\subsection{个人牺牲}

当没有机构付费时，创作者用自己的资源付费：储蓄、健康、人际关系、未来的收入能力。

Hickey 花了2.5年退休储蓄来构建 Clojure，因为"我无法在抽象层面说服人们接受我的想法，所以我不得不自己构建实现。"从自筹资金到可持续支持（Cognitect，后被 Nubank 收购）的间隔大约为8年。

Processing 经历了\textbf{11年}（2001--2012）没有专项资金。Processing Foundation 于2012年创建，正是因为志愿劳动无法再维持该项目。在最初十年中，"绝大多数代码由少数人志愿贡献时间编写，没有全职带薪开发者。"

Bret Victor 经历了两次接连的资金崩溃——SAP 从 CDG 撤出，然后 Sam Altman 在"他们搬进 Oakland 翻新建筑后仅数月"就终止了 HARC 的资助。Victor 的 Apple 储蓄可能弥补了这些空白。Dynamicland 最终成为 501(c)(3)，但其轨迹被机构背叛所定义。

\subsection{社区众筹}

Patreon、Open Collective、Ko-fi 和类似平台使用户能够直接资助创作者。金额通常不大。

Aaron 的 Sonic Pi 靠 Patreon 捐赠和付费演讲维持。Olivia Jack 的 Hydra 靠 Open Collective 贡献和200美元的微额资助运行。Alex McLean 的 TidalCycles 从个人 Ko-fi 过渡到 Open Collective。Shiffman 用 Coding Train YouTube 收入和 Patreon 补充他的 NYU 薪水。

模式是：众筹作为另一收入来源的补充。在我们调查的任何案例中，它都不能作为主要收入来维持创作者。拥有最多用户的工具（Processing、p5.js、Scratch）并非获得最多众筹的工具。

\subsection{意外之财}

2021年，Processing Foundation 从使用 Processing 和 p5.js 的生成艺术家那里收到了大约\textbf{1,000万美元}的 NFT/加密货币捐赠。这比 Foundation 前9年收到的所有资金总和还多。它使他们第一次能够雇用全职员工。

这一事件不可重复、不可预测、不可规划。它在一夜之间改变了一个有20年历史的项目。我们调查中没有其他创意工具经历过类似的意外之财。这不是一个模式；这是运气。

\subsection{完全没有模式}

一些创作者完全没有资金模式——甚至没有大学薪水的隐性补贴。他们靠残疾补助金、储蓄或什么都没有来构建。

Terry Davis 靠社会保障残疾金（约750--1,200美元/月）构建 TempleOS，住在父母家中。当他在2017年失去住所后，粉丝们送来物资，但他拒绝了他们提供的住所。他的 YouTube 频道被暂停；他的 PayPal 被他人入侵；即使是小额捐赠也无法到达他手中。他继续从公共图书馆发帖直至去世。该项目是公共领域的。他免费将其送出。

Goodiepal 的 \emph{El Camino del Hardcore}~\citep{goodiepal2012elcamino}——一份印在灰色硬纸板上的191页"激进计算机音乐"乐谱——从内部记录了这种模式的劳动条件。文本描述了被皇家丹麦音乐学院开除，作品"花了很久才成为现实"，以及"创作某些可能只在我自己可能有病的大脑中才有意义的东西"的焦虑。乐谱面向"一个可能的未来替代智能"——一个尚不存在且可能永远不会付费的受众。

"无模式"类别是终端情况。没有大学薪水或企业赞助者提供结构时，创作者的身体就成了基础设施。当身体因疾病、无家可归或死亡而失败时，项目要么成为公共领域的孤儿软件，要么完全消失。

\section{抹除问题}

Hernando Barrag\'an 于2003年在 IDII 作为硕士论文创建了 Wiring~\citep{barragan2004wiring}。他的导师 Massimo Banzi 和同事改编了 Wiring 的设计用于更便宜的硬件，并推出了 Arduino。Arduino 筹集了5,400万美元。Wiring 什么也没得到。多年来，Barrag\'an 没有获得公开认可。在2013年 Open Hardware Summit 上，一名 Arduino 团队成员承认他们"在承认 Wiring 方面做得不太好。"

这是最严峻的案例，但这种模式——学生创建工具，机构或导师获益——贯穿整个创意计算。Maeda 的学生从他的 Design By Numbers 创建了 Processing；Maeda 成为 RISD 校长，而 Processing 的创造者花了十年没有资金。

抹除并不总是有意的。它是结构性的：拥有机构权力的人（教授、导师、企业赞助者）处于有利位置，可以从没有权力的人（学生、博士后、独立开发者）创建的工具中获取价值。

\section{作者的立场}

\ac{} 自2021年以来一直在开发中~\citep{scudder2026ac}。作者包含这一节，是因为关于创意工具的学术论文中围绕资金的沉默延续了工具源于纯粹智力努力的神话。

作者于2021年离开了一个终身教职轨道的学术职位，全职开发 \ac{} 和 Whistlegraph 项目。从那以后，\ac{} 在没有机构支持的情况下构建——没有大学、企业或基金会的支持。详细的财务历史可应要求提供用于研究目的。

两种社区驱动的资金机制已经出现。第一，\texttt{give.aesthetic.computer}——一个直接捐赠页面，用户在此捐款以维持平台基础设施的运行。第二，\texttt{aesthetic.direct}——一个股权投资页面，向志同道合的投资者提供 Aesthetic Inc.（平台背后公司）的股份。少数个人投资者通过 SAFE 协议参与（"booted-by"模式，以平台的启动序列命名）。两种机制都没有产生机构级别的资金；两者都依赖于个人对项目的信念而非市场回报或资助周期。

本文中的每个其他工具都在机构保护伞下构建。Processing 有 MIT，后来有 UCLA。p5.js 有 UCLA。Sonic Pi 有 Cambridge。ChucK 有 Stanford。Pure Data 有 UCSD。甚至 Perl 也有 O'Reilly。\ac{} 没有这些。比较不是"有资助的工具 vs. 无资助的工具。"而是"在机构内构建的工具 vs. 在所有机构之外构建的工具。"

这很重要，因为 \ac{} 不是一个原型。它有注册用户群、数万次启动会话、数千个 KidLisp 程序和数百个用户发布的作品。作者的前一个项目 Whistlegraph 获得了260万 TikTok 粉丝。人们在该平台的设计之上构建并汲取灵感——提示符即乐器的隐喻、作品模型、社交平台上的生成艺术方法。该项目获得了机构认可（KADIST、SMK 丹麦国家美术馆、ZKM、Rhizome），并有一个每天使用它的社区。

平台是真实的。用户是真实的。资金不是。

\section{什么可能有效}

Eghbal 的 \emph{Roads and Bridges}~\citep{eghbal2016roads} 和 \emph{Working in Public}~\citep{eghbal2020working} 广泛记录了开源软件中的基础设施危机。创意计算工具面临同一危机的复合版本：它们是基础设施（被数百万人使用），它们是艺术（在文化上有价值但在商业上无价值），它们是教育（对教学法至关重要但没有以此身份获得资助）。

Czaplicki 的农业隐喻很恰当："在丰年积蓄以度过荒年。"但创意工具作者很少有丰年。Processing Foundation 的 NFT 意外之财是一代人一次的丰年。大多数创作者只经历荒年。

可能有效的模式：
\begin{itemize}
  \item \textbf{主权技术基金}：德国的 Sovereign Tech Fund 给了 p5.js 45万欧元。这种模式——政府资助开源基础设施——可以扩展到创意工具。
  \item \textbf{工人合作社}：Ramsey Nasser 的 EMMA Technology Cooperative（纽约4人合作社）提供了独自面对困境和企业雇佣之外的替代方案。
  \item \textbf{长期企业奖学金}：Matz 的28年 NaCl 奖学金是我们调查中最持久的安排。依赖创意工具的公司可以提供类似的安排。
  \item \textbf{维护津贴}：Ukeles 论证维护是一种创造性行为~\citep{ukeles1969manifesto}。为维护现有工具——而不仅仅是构建新工具——提供的津贴将解决最常见的失败点。
  \item \textbf{诚实的核算}：每篇关于创意工具的论文都应包含一节说明其开发是如何获得资助的。围绕资金的沉默延续了工具源于纯粹智力努力的神话，遮蔽了使其成为可能的物质条件。
\end{itemize}

\section{结论}

被数百万学生、艺术家和音乐家使用的创意计算工具，是由在大多数情况下没有因此获得报酬的人构建和维护的。他们的劳动被不重视它的大学补贴，被利益可能偏离的企业补贴，被会耗尽的个人储蓄补贴，以及被会恶化的健康补贴。这个领域没有资金模式；它有一堆应对策略。

本文没有解决这个问题。它为问题命名。这里调查的26位创作者——从 Wall 到 McCarthy 到 Puckette 到作者——共享一个共同条件：他们制造了别人需要的工具，而他们没有得到与这种需求相称的补偿。创建到获得资金之间的间隔以年计。代价以倦怠、裁员、疾病和消失来衡量。

问题不在于创意工具是否有价值。它们显然有价值。问题在于制造它们的人是否能继续下去。对一些人来说，答案是否。Davis 在靠残疾补助金构建了121,000行操作系统后死在铁轨旁。\_why 在被人肉搜索后删除了一切，从未收到过一分钱。Aaron 在 Cambridge 撤回支持后面临"财务生存"。Van Rossum 在29年的社区虐待后辞职。Wall 在失业和手术中继续开发 Perl。Goodiepal 因"交了错误类型的朋友"而被皇家学院开除。

模式是一致的：每一个被广泛使用的创意计算工具都在机构保护伞下构建——大学、企业或基金会——而工具自身的文档很少承认这一点。当保护伞被撤走时，创作者就暴露了。当一开始就没有保护伞时，创作者的身体就成了基础设施，而身体会失败。

本文的作者处于第五年，拥有活跃的用户群，没有机构支持。这不是抱怨；这是一个数据点。下一个数据点将取决于这个领域是发展出资金模式，还是继续依赖其创作者的身体。
