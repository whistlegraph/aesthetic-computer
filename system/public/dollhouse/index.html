Path A — One Room

Jeffrey fills a room with AC

I bring a device with HDMI out running a browser fullscreen. The projection is live software — not a video loop. It responds to time and input. Audience scans a QR and interacts from their phone.

Instruments:

notepat — Keyboard-driven musical instrument. Audience sees a visual piano reacting to notes in real time. Scan the QR, play the room from your phone. Keys light up. Sound fills the space. Multiple people can play at once.

> aesthetic.computer/notepat

clock — UTC-synchronized clock that plays generative melodies. Compact notation like clock cdefg loops melodies with waveform control, swing, parallel tracks, and tempo shifts. The room becomes a ticking musical object.

> aesthetic.computer/clock

kidlisp — Minimal Lisp for generative art. 118 built-in functions. Spinning cubes, procedural patterns, color fields, 3D forms. The QR takes you to kidlisp.com where you can live-edit the code running on the wall.

> kidlisp.com

Slideshows:

merryo loops through pieces with custom timing — each "slide" is a live running program, not an image. mo is shorthand for uniform timing.

> merryo:8-notepat:8-clock:8-kidlisp 8s each, loops forever > mo.1:tone:shape:line 100ms rapid cycle

I'd likely combine these — KidLisp visuals as base layer, notepat or clock for sound, sequenced via merryo. Or a custom piece for DOLLHOUSE. The key: live software, not a recording. The room is awake.

I have a box that runs all of this at 4K over HDMI. Only requirements: power + projection surface.

Path B — Full Infrastructure

AC powers the whole show

If you want to go bigger: AC can serve as the technical backbone for all 30 rooms, the anchor screen, and audience interaction.

roomsEach room runs a browser at aesthetic.computer/@artist/room
anchortop.kidlisp.com — TV slideshow system, 4K HDMI
slideshowmerryo / mo — auto-cycle pieces on any screen
audienceQR → browser → interactive piece, no app needed
livePerformer drives visuals from phone/tablet in real-time
syncSession server for coordination across rooms

Audience flow:

See room → scan QR → phone opens piece → interact / edit / play

Every room gets a QR code. Scanning opens that room's piece on your phone. Play it, remix it, or edit the code running on the wall.

Onboarding artists:

  • Code experience → full JS API (graphics, audio, network)
  • No code → KidLisp (a few lines = complex generative visuals)
  • Publish and iterate from a browser, no dev env needed

Anchor screen:

top.kidlisp.com cycles through curated pieces from all rooms using merryo sequencing. 4K box with HDMI out.

AC outputs rectangular frames — not projection-mapped surfaces. Pair with MadMapper/Resolume for non-rectangular projection. Audio is per-device; individual rooms need their own output chain.

Both paths work independently or together. Happy to fill one room and make it sing, or help wire up the whole building.
