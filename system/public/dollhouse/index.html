A proposal for Turbo Cheap — Frieze LA opening weekend

Hey John — this is a rundown of what Aesthetic Computer can do for DOLLHOUSE at Turbo Cheap. Two paths below: one where I fill a single room as an artist, and one where AC powers the whole show infrastructure. Both can coexist. Click any pink command to open a live preview.

Path A — One Room Jeffrey fills a room with AC

I bring a device with HDMI out running a browser at full screen. The projection is a live, running program — not a video loop. It responds to time, to input, and audience members can interact from their phones by scanning a QR code on or near the room.

Here are the instruments I'd pull from:

Instrument

Notepat

A keyboard-driven musical instrument. The audience sees a visual piano interface reacting to notes in real time. Someone scans the QR, and now they're playing the room from their phone. Keys light up. Sound fills the space. Multiple people can play at once.

notepat
> aesthetic.computer/notepat
Instrument

Clock

A visual UTC-synchronized clock that plays generative melodies. Specify melodies in compact notation — clock cdefg — and it loops them with waveform control, swing, parallel tracks, and tempo shifts. The room becomes a ticking musical object, different every time you listen.

clock
> aesthetic.computer/clock
Language

KidLisp

A minimal Lisp dialect for generative art — 118 built-in functions across 12 categories. Spinning cubes, procedural patterns, color fields, 3D forms. The projection runs a KidLisp program, and the QR code takes you to kidlisp.com where you can live-edit the code and see your changes on the wall.

kidlisp.com
> kidlisp.com
Slideshow

Merryo & Mo

AC has built-in slideshow sequencers. merryo loops through a sequence of pieces with custom timing — each "slide" is a live running program, not an image. mo is shorthand for uniform timing across all pieces.

Examples:
merryo:8-notepat:8-clock:8-kidlisp
Loop notepat, clock, kidlisp — 8 seconds each
mo.1:tone:shape:line
Rapid 100ms cycling between pieces

This means a room projection can be a curated sequence of live programs, auto-cycling on loop. Or the anchor screen can rotate through every artist's room as a living slideshow — via top.kidlisp.com.

The Combination

I'd likely combine these — a room that shifts between modes or layers them. Generative visuals in KidLisp as the base layer, with notepat or clock providing the sonic dimension. Or a custom piece built specifically for DOLLHOUSE. Or a merryo slideshow sequencing all of them on a timer.

The key thing: it's live software, not a recording. The room is awake.

I have a box that runs this at 4K over HDMI. I can also bring small test devices. The only requirement is power and a projection surface (or screen).

Path B — Full Infrastructure AC powers the whole show

If you want to go bigger: Aesthetic Computer can serve as the technical backbone for all 30 rooms, the anchor screen, and the audience interaction layer.

30 Rooms, 30 Pieces

Each room runs an AC "piece" — a small program at a unique URL. Artists create their visuals in JavaScript or KidLisp. Every room is just a browser tab on a connected device. No special software. No app installs for artists or audience.

Audience Flow

See room → Scan QR → Phone opens piece → Interact / edit / play

Every room can have a QR code. Scanning it opens that room's piece on the audience member's phone. They can play it, remix it, or in the case of KidLisp pieces, actually edit the code that's running on the wall. Spectators become participants.

Artist Onboarding

  • Artists with code experience use the full JavaScript API — graphics, audio, network, UI
  • Artists without code use KidLisp — a few lines produce complex generative visuals
  • A blank template generates in one command: npm run new room-name
  • Artists publish and iterate from a browser — no dev environment needed

The Anchor Screen

top.kidlisp.com is a purpose-built slideshow/display system. It can cycle through curated pieces from all 30 rooms using merryo sequencing, hold on a single live visual, or run its own program. Driven by a 4K box with HDMI out.

Live Performance

Pieces respond to real-time input — touch, keyboard, microphone, pen. A performer could drive the anchor screen live from a phone or tablet. The session server supports multiplayer sync, so you could have a moment where every audience member's phone feeds into one shared display.

What AC doesn't do: It outputs rectangular frames — not projection-mapped surfaces. For non-rectangular projection you'd pair it with a media server (MadMapper, Resolume, etc.). Audio routing is per-device, so individual room audio needs its own output chain.

Both paths work independently or together. I'm happy to fill one room and make it sing, or help wire up the whole building. Let's talk about what fits.
