§Pitch

The AC Papermill has been writing things down for five years. Forty-four arXiv-format papers across the system, the language, the network, the music. Fourteen organizational dossiers — Rhizome, SFPC, Eyebeam, Recurse, Internet Archive, Mellon, Pioneer Works, NEW INC, Studio Museum, HathiTrust, The Kitchen, Machine Project, Heavy Manners, MicroVision — each with structured 990-PF / IRS XML data tables in data/ alongside the prose. A bound Reader collecting all of it. Translations into four languages. 137 MB on disk.

At a certain point a corpus this sourced, this footnoted, with this many cross-references, stops being a stack of papers you read sequentially and starts being a knowledge base you query. The Project Space is the night the platter hosts a reference desk.

A new-media expert system, in the classical sense.

Mid-twentieth-century AI's first real public win was the expert system — DENDRAL diagnosed mass spectra, MYCIN diagnosed bacterial infections, R1 configured VAX minicomputers. The recipe: a knowledge base hand-built from domain experts, an inference engine that ran rules over it, and an explanation facility that traced every answer back to the rules and facts that produced it. Expert systems were narrow and footnoted; large language models are general and unfootnoted. The interesting territory in 2026 is the narrow, sourced middle ground.

The AC papermill, because every paper is built from sourced material, every dossier ships a data/ README that lists solid / soft / missing, and every cover image's prompt is research-cited in cover-sources.md, satisfies the classical expert-system demands more than most generative systems do: knowledge base, inference engine (a small local retrieval step + Claude Code), and an explanation facility (the answer comes back with the file paths, page numbers, and dataset rows it used). The booth at NYABF is where that machine sits down at a public reference desk.

What the visitor does.

They sit down, type a question in their own words, and watch the platter answer. The answer is footnoted: each claim points to a paper, a page, and (for dossier claims) a row of structured data. The answer prints as a single-sheet card — query at top, answer mid, footnote ledger at bottom, QR linking to a permanent URL where the same query is hosted online. The visitor walks away with the card, the cited papers from the wall, and a copy of the Reader.

The booth is quiet on purpose. No daily zines, no concerts, no flashing of laptops. A wall of papers; one laptop; one printer; the visitor's question.

Why this works as a Project Space.

NYABF Project Spaces frame a publication or printed-matter practice rather than just selling books from a table. Aesthetic Computer's printed matter is a knowledge base, and the booth is the demonstration that it has accumulated to the threshold of being one. The cultural object is the platter; the demonstration is that it can answer back.

§Four Registers

Knowledge base

137 MB on disk. 44 sourced arXiv papers, 14 audit-trailed dossiers, 1 bound Reader, 3 sub-platters, in 4 languages. Everything has a citation; every dossier has a data/.

Inference engine

A small local retrieval step over the platter's .tex / .md / .bib / .csv files, paired with Claude Code as the answering shell. Wifi-up at the booth; a fallback embedding-search index is cached on the disk.

Explanation facility

Every answer comes back with file paths, page numbers, and (for dossiers) row-level data citations. The card the visitor takes home shows its work.

Reference desk

One laptop. One printer. One artist behind the desk. A wall of papers free to take. A visitor's question, in their own words, gets a footnoted answer they can hold.

§The Expert-System Lineagespecific, not canonical

DENDRAL · Stanford
Feigenbaum, Buchanan, Lederberg (1965–). The first widely cited expert system; diagnosed mass spectra. The recipe was knowledge base + inference engine + explanation — and the platter satisfies all three.
MYCIN · Shortliffe
1972–. Bacterial-infection diagnosis with explicit certainty factors. Precedent for narrow, footnoted machine answers in a domain where being wrong was expensive.
R1 / XCON · DEC
McDermott, 1980. Configured VAX minicomputers; the first expert system to make money. Proof that a corpus with rules can do real, narrow work better than a generalist.
Cyc · Lenat
1984–. The most ambitious knowledge-base project of the 20th century. The platter is the opposite scope (140 MB, hand-curated, single-domain) but the same posture — encode it so a machine can answer with it.
CLIPS · NASA
1985–. Open-source expert-system shell; still ships in classrooms. The precedent for the shell-plus-knowledge-base separation the booth runs.
Wiki / WikiData
2001 / 2012. Crowdsourced, footnoted knowledge as a public infrastructure. The platter is the artist-run version — narrower, single-author, more opinionated, sourced the same way.
RAG (retrieval-augmented generation)
2020–. The contemporary recipe: local corpus + retrieval + LLM. The platter qualifies as the corpus; what's new at the booth is that it's been pressed into print first, then read back as a machine query.
