üîß Remaining Issues to Fix

1 CelShadePP_Mat ‚Äî GBuffer Not Available on Mobile CRITICAL

This is the main issue causing the dark scene. The cel-shading post-process material is trying to access GBuffer textures (WorldNormal, BaseColor, etc.) which don't exist on iOS because mobile uses forward rendering, not deferred.

Error from Cook Log

(Node SceneTexture) GBuffer scene textures not available with forward shading (platform id 11).
(Node SceneTexture) GBuffer scene textures not available in post-processing with mobile shading (platform id 11).

LogMaterial: Warning: Failed to compile Material for platform SF_METAL_ES3_1_IOS, Default Material will be used in game.

Root Cause

The material graph contains SceneTexture nodes set to access GBuffer data like:

  • WorldNormal ‚Äî Not available on mobile
  • BaseColor ‚Äî Not available on mobile
  • Metallic/Specular/Roughness ‚Äî Not available on mobile

Fix in Unreal Editor

  1. Open Content/Developers/yueha/FX/CelShadePP_Mat
  2. Find all SceneTexture nodes in the material graph
  3. For each node accessing GBuffer data, either:
    • Remove it and use a simpler approach
    • Wrap it in a Feature Level Switch node with a mobile fallback
    • Use only mobile-compatible inputs: PostProcessInput0, SceneDepth, CustomDepth
  4. Test by previewing in ES3.1 Preview Mode (Settings ‚Üí Preview Rendering Level)
  5. Save, submit to P4, and trigger new iOS cook

Alternative: Disable for iOS

If cel-shading isn't critical for mobile, you can disable it:

  1. Open the Post Process Volume that uses CelShadePP_Mat
  2. Add a branch that checks platform: Is Platform Mobile
  3. Skip applying this post-process on mobile platforms
2 CelShadeRock ‚Äî Missing Material Function HIGH

Content/Developers/yueha/Migrate/Beach/Rock_Pack/Material/MI/CelShadeRock.uasset

Fix in Unreal Editor

  1. Open Content/Developers/yueha/Migrate/Beach/Rock_Pack/Material/MI/CelShadeRock
  2. Look for any Material Function nodes with a red error state
  3. Check if the referenced function exists and is in the correct location
  4. If the function was deleted/moved, re-link it or recreate it
  5. Ensure all referenced functions are set to be cooked (Used with settings)
  6. Save and submit to P4
