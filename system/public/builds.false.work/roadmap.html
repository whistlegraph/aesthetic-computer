Overview

This roadmap outlines the development of a story collection system for false.work builds. Players can write stories about their experience playing builds across different platforms. These stories help track what works, what breaks, and what could be improved.

Development Phases

Phase 1: Foundation Done

Basic infrastructure for build distribution.

  • Build download system for Windows, macOS, iOS
  • Version tracking with timestamps
  • Platform-specific installation instructions
  • Dark mode support
  • Mobile-responsive design

Phase 2: Stories Done

Players can write stories about their experience playing builds.

  • Story submission form
  • MongoDB storage for stories
  • API endpoints (GET/POST stories)
  • Three-category feedback: ğŸŠ Fruit (what worked), ğŸ¥— Side (what broke), ğŸ’¡ Ideas (bigger thoughts)
  • Story cards nested under builds in feed view
  • Individual story detail pages
  • Tester name and timestamp display
  • Mobile-optimized touch targets

Phase 3: Build Management Active

Tools for uploading and managing builds.

  • Manual build file hosting
  • Admin upload interface
  • Automatic version detection
  • Build notes / changelog per version
  • Archive old builds

Phase 4: Analytics Planned

Insights from stories and usage patterns.

  • Story trends over time
  • Common issues across builds
  • Per-build story summary
  • Export stories as CSV/JSON

Data Model

Story Document

{
  "_id": ObjectId,
  "timestamp": "2025-01-15T14:30:00.000Z",
  "build": "windows-x64-20250115",
  "tester": "Player Name",
  "fruit": "Things that worked well...",
  "side": "Things that broke or felt off...",
  "ideas": "Bigger thoughts and suggestions..."
}

Stories are stored in the false.work-reports collection. Each story captures feedback in three categories using food metaphors: fruit (the good stuff), sides (the rough edges), and ideas (the bigger picture).

API Endpoints

GET /.netlify/functions/stories

Returns all stories, newest first.

{
  "stories": [
    {
      "_id": "...",
      "timestamp": "...",
      "build": "...",
      "tester": "...",
      "fruit": "...",
      "side": "...",
      "ideas": "..."
    }
  ]
}

POST /.netlify/functions/stories

Submit a new story about playing a build.

// Request body:
{
  "build": "windows-x64-20250115",
  "tester": "Player Name",
  "fruit": "What worked...",
  "side": "What broke...",
  "ideas": "Bigger thoughts..."
}

// Response:
{
  "success": true,
  "id": "..."
}
